ChatGPT에 질문을 던지는 것과 AI 비서를 운용하는 것은 다릅니다.

전자는 즉흥적입니다. 그때그때 원하는 걸 타이핑하고 답을 받습니다. 후자는 훈련된 루틴입니다. AI가 내 업무 맥락을 알고, 내가 하는 일의 규칙을 따르고, 반복 작업을 자동으로 처리합니다.

스킬(Skill) 시스템은 그 간격을 메우는 방법입니다.

---

## 스킬이란 무엇인가

Claude Code에서 스킬은 `.claude/skills/` 폴더에 저장된 마크다운 파일입니다. 사람으로 치면 "SOP(표준 운영 절차)"입니다. AI에게 "이런 상황에서는 이렇게 해라"를 문서화한 것이고, 트리거 조건이 맞으면 자동으로 활성화됩니다.

예를 들어 이런 스킬이 가능합니다.

- `/blog-draft` — 주제를 주면 블로그 규칙에 맞는 초안을 생성
- `/youtube-topic-finder` — 웹 검색으로 유튜브 에피소드 주제를 발굴하고 Linear에 등록
- `/blog-publish` — 초안 검증 후 적절한 폴더로 이동, git push까지

"블로그 초안 써줘"라고 말하는 것과, `/blog-draft 오늘 주제`를 치는 것은 결과가 다릅니다. 전자는 매번 규칙을 다시 설명해야 하고, 후자는 스킬 파일에 규칙이 내장되어 있어서 매번 같은 기준으로 동작합니다.

---

## K-스킬이 왜 필요한가

"AI 스킬"이라는 개념은 영어권 사용자 기준으로 설계된 것이 많습니다. 한국어로 작업하는 환경에는 맞지 않는 부분이 있습니다.

한국 사용자가 실제로 하는 일은 다릅니다.

- 블로그를 한국어로 씁니다. 프론트매터 규칙, 존댓말 톤, SEO 키워드가 다릅니다.
- Kakao 오픈채팅, Naver 카페, 클리앙 같은 플랫폼에서 정보를 수집합니다.
- Linear, Notion 한국어 워크스페이스를 씁니다.
- 팟캐스트 스크립트를 2인 대화체 한국어로 만듭니다.

이 작업들을 위한 스킬을 한국어 환경에 맞게 만든 것이 K-스킬입니다. 영어 스킬을 번역한 것이 아니라, 한국 사용자의 실제 워크플로우를 중심으로 설계한 스킬입니다.

## 스킬 시스템 구조

스킬 파일의 구조는 단순합니다.

```
.claude/
  skills/
    blog-draft/
      SKILL.md        ← 스킬 명세
    blog-publish/
      SKILL.md
    youtube-topic-finder/
      SKILL.md
```

각 SKILL.md에는 다음이 들어갑니다.

1. **트리거 조건** — 언제 이 스킬을 쓸 것인가
2. **사용할 도구** — WebSearch, Linear API, git 등
3. **워크플로우** — 단계별로 무엇을 어떻게 할 것인가
4. **금지 사항** — 절대 하면 안 되는 것

AI는 이 파일을 읽고, 트리거 조건이 맞으면 자동으로 그 흐름을 따릅니다. 사용자는 매번 "이렇게 해줘"를 설명하지 않아도 됩니다.

## 실전 설치와 사용

Claude Code가 설치된 환경에서 스킬을 추가하는 방법은 두 가지입니다.

**방법 1: 직접 작성**

```
mkdir -p .claude/skills/my-skill
touch .claude/skills/my-skill/SKILL.md
```

SKILL.md에 워크플로우를 작성합니다. 영어로 써도 되고, 한국어로 써도 됩니다. Claude는 둘 다 읽습니다.

**방법 2: skill-creator 스킬 사용**

이미 Claude Code 환경이 갖춰져 있다면 `/skill-creator`를 사용하면 됩니다. 원하는 스킬을 설명하면 Claude가 SKILL.md 파일을 직접 생성해줍니다.

스킬이 설치된 후에는 두 가지 방법으로 실행합니다.

- **명시적 실행**: `/스킬명`으로 직접 호출
- **암묵적 트리거**: 대화 중 스킬 설명에 맞는 요청이 오면 자동 활성화

---

## 한국 업무용 K-스킬 예시

실제로 한국 환경에서 유용한 스킬들입니다.

**콘텐츠 제작팀**
- 블로그 초안 자동 생성 (한국어 SEO 규칙 포함)
- 유튜브 주제 발굴 + Linear 등록
- 팟캐스트 스크립트 2인 대화체 작성

**리서치·기획팀**
- 클리앙·뉴스하다 트렌드 수집 + 요약
- 논문 리뷰 → 블로그 포스트 변환
- Linear Status Update → 문서 자동 생성

**개인 지식 관리**
- Obsidian 볼트 자동 정리
- 웹 클리핑 → 핵심 요약 메모
- 메모리 인덱스 자동 업데이트

## 직접 만들 수 있다

스킬은 코딩 지식 없이도 만들 수 있습니다. 핵심은 "내가 이 작업을 할 때 어떤 순서로 무엇을 확인하는가"를 글로 쓰는 것입니다.

예를 들어 "주간 팀 리포트를 자동으로 만들고 싶다"면 이렇게 생각합니다.

1. 어디서 데이터를 가져오는가 (Linear, Notion, Slack)
2. 어떤 형식으로 정리하는가 (표, 글머리표, 서사형)
3. 어디에 저장하는가 (Notion 페이지, 이메일, Slack 메시지)
4. 절대 틀리면 안 되는 것은 무엇인가

이 네 가지를 SKILL.md에 쓰면 스킬이 됩니다. `/skill-creator`에게 설명하면 파일 형식까지 맞춰서 만들어줍니다.

AI가 "뭔가 해주는" 시대에서 "내 방식대로 해주는" 시대로 넘어가고 있습니다. 스킬은 그 전환점입니다.
