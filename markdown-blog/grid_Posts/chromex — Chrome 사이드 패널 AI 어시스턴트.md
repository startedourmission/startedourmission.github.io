브라우저 확장으로 AI를 쓰는 방식은 대부분 비슷합니다. 페이지에 플로팅 버튼을 띄우거나, 드래그한 텍스트를 복사해서 GPT 창을 새로 열거나. 컨텍스트를 직접 전달하는 게 아니라 사용자가 복붙으로 이어붙이는 구조입니다.

chromex는 이 문제를 다르게 접근합니다. Chrome 사이드 패널에 붙어서 현재 페이지 내용, 열린 탭 목록, 스크린샷을 자동으로 AI에게 넘깁니다. 사용자가 복붙할 게 없습니다.

## 구조: 왜 로컬 서버가 필요한가

chromex의 아키텍처는 네 단계입니다.

```
Chrome Extension → Native Messaging Host → Local Bridge → Codex app-server
```

Chrome 확장은 Native Messaging Host를 통해 로컬 프로세스와 통신합니다. 이 로컬 브리지가 실제로 [[OpenAI]] Codex API를 호출합니다.

이 구조를 선택한 이유가 중요합니다. **API 키가 Chrome 확장 스토리지에 저장되지 않습니다.** Chrome 익스텐션은 개발자 도구로 스토리지 내용을 쉽게 볼 수 있고, 악성 확장이 다른 익스텐션의 스토리지를 읽는 공격도 가능합니다. chromex는 API 키를 로컬 Codex 앱 서버에서만 관리하고, 확장은 그 서버와만 통신합니다.

세션 히스토리도 기본적으로 메모리에만 유지됩니다. 브라우저를 닫으면 대화 기록이 사라집니다.

## 주요 기능

**페이지 컨텍스트 자동 주입**

현재 보고 있는 페이지의 텍스트를 자동으로 컨텍스트로 넘깁니다. "이 페이지 요약해줘"라고 입력하면 URL만 던지는 게 아니라 실제 내용을 AI가 바로 봅니다.

**탭 선택기**

`@` 키를 누르면 열린 탭 목록이 나옵니다. 여러 탭을 한 번에 컨텍스트로 추가할 수 있습니다. 여러 문서를 비교하거나 여러 탭 내용을 합쳐서 정리할 때 유용합니다.

**YouTube 어댑터**

YouTube 페이지에서는 현재 재생 위치의 타임스탬프와 자막을 컨텍스트로 씁니다. "방금 나온 개념 설명해줘"가 실제로 작동합니다.

**파일 업로드**

PDF, DOCX, CSV, XLSX를 직접 올릴 수 있습니다. 페이지 컨텍스트와 파일을 함께 쓸 수 있습니다.

**음성 입력**

마이크 버튼으로 음성 입력을 지원합니다. 텍스트 입력과 병행 가능합니다.

**이미지 편집**

스크린샷을 찍거나 이미지를 붙여넣고 직접 그 위에 주석을 달 수 있습니다. 편집한 이미지를 컨텍스트로 전달합니다.

**사이트별 자동 제안**

YouTube, 뉴스, 연구 사이트, 한국 업무용 서비스(잔디, 카카오워크, 네이버웍스) 등 사이트 유형을 감지해서 관련 프롬프트 제안을 보여줍니다.

**Codex 스킬**

로컬 `.codex/skills/*/SKILL.md` 파일에 커스텀 스킬을 정의할 수 있습니다. [[MCP]]와 유사하게, 특정 작업 패턴을 미리 정의해두고 불러쓰는 방식입니다.

## 설치 방법

Chrome 웹 스토어에는 없습니다. 직접 빌드해야 합니다.

```bash
git clone https://github.com/GENEXIS-AI/chromex
cd chromex
npm install
npm run build
node install-native-host.mjs  # Native Messaging Host 등록
```

빌드 후 Chrome에서 `chrome://extensions` → 개발자 모드 → "압축 해제된 확장 프로그램 로드" → `dist/` 폴더 선택.

로컬 Codex 앱 서버(`codex-app-server`)도 별도로 실행해야 합니다. 이 서버가 실제 API 호출을 처리합니다.

## 어떤 상황에 쓸 만한가

페이지 내용을 직접 AI에게 넘기는 구조가 핵심 강점입니다. 롱폼 문서를 읽으면서 모르는 개념을 그 자리에서 물어보거나, 여러 탭을 열어두고 비교 분석하거나, YouTube 영상 보면서 내용 질문하는 용도에 맞습니다.

API 키를 로컬에서만 관리한다는 보안 설계도 실용적입니다. 회사 기밀이 담긴 문서를 다루는 상황이라면 API 키 유출 리스크를 줄이는 게 중요합니다.

단, 설치 허들이 있습니다. 빌드 환경 세팅과 네이티브 호스트 등록이 필요합니다. CLI에 익숙하지 않다면 진입이 쉽지 않습니다.
