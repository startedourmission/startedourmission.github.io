OpenAI가 2026년 6월 4일 ChatGPT의 메모리를 새로 갈아끼웠습니다. 이름이 Dreaming V3입니다. 그동안 사용자가 직접 관리하던 "저장된 메모리" 목록을, 수년치 대화를 가로질러 사용자에 대한 기억을 알아서 갱신하는 비동기 백그라운드 합성 프로세스로 바꿨습니다.

핵심은 두 가지입니다. 하나는 사용자가 더 이상 메모리를 손으로 큐레이션하지 않아도 된다는 것, 다른 하나는 이 일을 처리하는 연산이 5배 줄어서 Plus와 Pro를 넘어 무료 티어까지 풀 수 있게 됐다는 것입니다. 그리고 그 대가로 따라오는 프라이버시 비용이 있습니다.

## 무엇이 바뀌었나

이전 ChatGPT 메모리는 두 갈래였습니다. 사용자가 명시적으로 저장하거나 ChatGPT가 알아서 적어 두는 "저장된 메모리" 목록이 하나, 대화 시점에 과거 채팅에서 관련 조각을 끌어오는 검색 방식이 다른 하나였습니다. 목록은 사용자가 직접 들여다보고 지울 수 있었지만, 그만큼 손이 많이 갔고 시간이 지나면 낡은 항목이 쌓였습니다.

Dreaming V3는 이 구조를 단일 비동기 백그라운드 합성 프로세스로 통합했습니다. OpenAI는 백그라운드에서 메모리를 정리하는 발상 자체는 2025년 4월 Dreaming V0부터 시도했고, 이번이 그 계보의 세 번째 정식 세대라고 설명합니다.

- 출시일: 2026년 6월 4일
- 우선 대상: 미국의 Plus, Pro 구독자
- 확대 예정: Free, Go 티어 및 미국 외 지역 (수주 내)

## 꿈꾸기의 실제 메커니즘

"꿈꾸기"라는 비유 뒤에 있는 구조는 의외로 단순합니다. 메모리 상태가 대화 로그 안에 들어 있지 않다는 점이 핵심입니다. ChatGPT가 사용자에 대해 아는 내용은 대화 기록과 분리된 별도 데이터 레이어에 보관되고, 추론할 때 시스템 프롬프트에 주입됩니다.

이 별도 레이어를 백그라운드 프로세스가 비동기로 갱신합니다. 여러 대화를 동시에 읽고, 자연스럽게 흘러나온 맥락을 알아서 포착하고, 상황이 바뀌면 기존 기억을 다시 씁니다. OpenAI가 든 예시가 이 동작을 잘 보여줍니다.

> "7월에 싱가포르에 간다"는 메모리가, 여행이 끝나면 "2026년 7월에 싱가포르에 다녀왔다"로 사용자 개입 없이 스스로 갱신됩니다.

시점이 지나면 기록도 따라 늙는다는 것, 이게 기존 저장 목록과 가장 다른 점입니다. 목록 방식에서는 "7월에 싱가포르 간다"가 8월에도 그대로 남아 어색해졌습니다.

## RAG, 저장 목록과 무엇이 다른가

| 방식 | 메모리 형성 시점 | 갱신 | 사용자 관리 |
|------|------------------|------|-------------|
| 저장된 메모리 목록 | 명시적 저장 시 | 수동 | 목록 직접 편집 |
| 대화 시점 RAG 검색 | 추론 시 즉석 검색 | 자동(원본 기준) | 어려움 |
| Dreaming V3 | 백그라운드 비동기 합성 | 자동, 시점 반영 | 요약 페이지로 검토 |

RAG는 매번 과거 대화에서 관련 조각을 찾아오지만 원본이 낡으면 낡은 채로 끌려옵니다. 저장 목록은 깔끔하지만 사람이 계속 손봐야 합니다. Dreaming V3는 둘 사이를 메우려는 시도입니다. 백그라운드에서 여러 대화를 종합해 하나의 일관된 기억 상태를 유지하고, 시간이 지나면 그 상태를 다시 씁니다.

## 연산 5배 절감과 무료 티어 확대

이번 발표에서 OpenAI가 가장 강조한 숫자는 연산 효율입니다. 메모리 기능을 서빙하는 데 드는 연산을 5배 줄였고, 그 덕분에 Plus와 Pro에 묶여 있던 기능을 무료 티어까지 확대할 여지가 생겼다고 밝혔습니다.

OpenAI가 공개한 내부 평가 수치는 다음과 같습니다.

| 지표 | 2025년 시스템 | 2026년 Dreaming V3 |
|------|----------------|---------------------|
| 사실 회상 | 67.9% | 82.8% |
| 선호 준수 | - | 71.3% |
| 시간 민감 정확도 | - | 75.1% |

사실 회상이 67.9%에서 82.8%로 올랐다는 수치는 외부 보도에서도 동일하게 확인됩니다. 선호 준수와 시간 민감 정확도는 OpenAI 발표 자료 기준입니다.

## 프라이버시와 감사 추적 문제

자동 합성에는 분명한 트레이드오프가 따릅니다. 어시스턴트가 사용자에 대해 사용자 본인이 기억하는 것보다 더 많이, 더 정리된 형태로 알게 됩니다. 문제는 이걸 통제하는 수단이 토글 하나처럼 간단하지 않다는 데 있습니다.

OpenAI는 메모리 요약 페이지에서 무엇을 기억하고 있는지 검토하고 수정할 수 있다고 안내합니다. 다만 백그라운드 프로세스가 어떤 대화에서 어떤 결론을 끌어냈는지 추적하는 감사 추적(audit trail)이 제한적이라는 지적이 프라이버시 연구자들 사이에서 나옵니다. 기억이 어디서 왔는지 되짚기 어려우면, 잘못 합성된 기억을 발견하고 바로잡는 일도 그만큼 어려워집니다.

## 정리

Dreaming V3는 메모리를 사람이 관리하는 목록에서 모델이 알아서 합성하는 상태로 옮기는 변화입니다. 연산을 5배 줄여 무료 티어까지 개인화를 확대할 길을 열었고, 시간이 지나면 기억도 함께 늙게 만들어 낡은 메모리 문제를 풀었습니다.

대가는 통제권입니다. 어시스턴트가 나에 대해 더 많이 알수록, 그 앎이 어디서 왔고 어떻게 고치는지가 중요해집니다. 에이전트가 사용자를 대신해 더 많은 일을 하게 될수록 이 질문은 더 무거워질 겁니다. 개인화의 정확도와 감사 가능성을 어떻게 같이 끌고 갈지가 다음 과제입니다.
