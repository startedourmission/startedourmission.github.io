> 이 글은 [stablyai/orca](https://github.com/stablyai/orca) 리포지터리와 [공식 사이트](https://www.onorca.dev/)를 참고하여 작성했습니다.

코딩 에이전트를 하나만 쓰는 시대가 끝나고 있습니다. 이제 Claude Code, Codex, Grok CLI를 동시에 돌리고 결과를 비교해서 제일 나은 코드를 골라 쓰는 방식이 등장했습니다. YC W22 스타트업 stablyai가 만든 오픈소스 툴 Orca가 그 역할을 합니다.

## ADE가 무엇인가

ADE는 Agent Development Environment(에이전트 개발 환경)의 약자입니다. 기존 IDE(통합 개발 환경)가 코드 작성·실행·디버그를 한 공간에서 처리했다면, ADE는 여러 AI 에이전트의 실행·출력·병합을 한 공간에서 처리합니다.

단일 에이전트 도구와의 차이는 명확합니다. Cursor나 Devin은 하나의 에이전트가 순차적으로 작업합니다. 에이전트가 코드를 짜는 동안 기다려야 하고, 결과가 마음에 들지 않으면 처음부터 다시 시작해야 합니다. Orca는 이 대기 시간을 병렬화합니다. 하나의 프롬프트를 여러 에이전트에 동시에 던지고, 각 에이전트가 독립된 환경에서 작업한 결과를 한꺼번에 비교합니다.

## 병렬 에이전트 실행 방식

Orca의 핵심 메커니즘은 Git worktree 분기입니다. 프롬프트 하나를 입력하면 Orca가 N개의 격리된 git worktree를 만들고, 각 worktree에서 다른 에이전트를 실행합니다.

작동 흐름은 다음과 같습니다.

1. 사용자가 프롬프트를 입력합니다. ("인증 미들웨어에 rate limiting 추가해줘")
2. Orca가 선택된 에이전트 수(예: 3개)만큼 worktree를 fork합니다.
3. Claude Code, Codex, OpenCode가 각자 독립된 환경에서 동시에 구현을 시작합니다.
4. 각 에이전트의 터미널 출력이 탭 형태로 실시간 스트리밍됩니다.
5. 모든 에이전트가 완료되면 side-by-side diff 비교 화면이 열립니다.
6. 사용자가 각 구현에서 원하는 부분을 hunk 단위로 골라 최종 merge worktree에 가져옵니다.

지원하는 에이전트는 30종 이상입니다. Claude Code, OpenAI Codex, Cursor CLI, Grok, Copilot CLI, OpenCode, Devin, Goose, Cline 등이 포함됩니다. 사용자 본인의 API 구독을 그대로 쓰므로 추가 비용이 없습니다.

## 단일 에이전트 대비 실질 차이

같은 문제를 Claude Code 하나에 맡기는 것과 Claude Code + Codex + OpenCode 셋에 동시에 맡기는 것은 결과가 다릅니다.

첫째, 에이전트마다 코드 생성 경향이 다릅니다. Claude Code는 인라인 타입 힌트와 함수 분리를 선호하고, Codex는 좀 더 함축적인 스타일로 구현하는 경향이 있습니다. 같은 스펙도 구현 전략이 달라서 각각의 결과가 참고 자료가 됩니다.

둘째, 병렬 실행으로 탐색 속도가 높아집니다. 하나의 구현이 막혀 있는 동안 다른 에이전트가 다른 방향으로 시도하고 있습니다. "잘못된 방향에 시간을 낭비했다"는 경우가 줄어듭니다.

Orca Discord 커뮤니티에서 집계한 사용자 데이터(2026년 7월)에 따르면, 단일 에이전트로 하루 5~10건을 처리하던 개발자들이 Orca 병렬 플릿으로 30~50건을 처리한다고 합니다. 공식 검증된 수치는 아니지만, 구조적으로 납득이 가는 차이입니다.

## 주요 기능

**모바일 동반 앱.** iOS와 Android 앱이 있습니다. 이동 중에 에이전트 실행 상황을 모니터링하고 간단한 지시를 내릴 수 있습니다. 서버에서 돌아가는 에이전트를 핸드폰으로 관제하는 개념입니다.

**Design Mode.** 브라우저와 통합된 시각 편집 모드입니다. 에이전트가 만든 UI 컴포넌트를 브라우저에서 직접 보면서 수정을 지시할 수 있습니다.

**SSH 원격 워크트리.** 로컬이 아닌 원격 서버의 환경에서 에이전트를 실행할 수 있습니다. GPU가 필요한 학습 스크립트나 프로덕션에 가까운 환경이 필요한 작업에 쓸 수 있습니다.

**GitHub + Linear 통합.** 이슈를 Orca에 직접 가져와서 에이전트에 던질 수 있습니다. 에이전트가 처리한 결과를 PR로 바로 올리는 워크플로우도 지원합니다.

**Orca CLI.** 에이전트가 IDE를 역으로 제어하는 인터페이스입니다. 에이전트가 Orca 탭을 열거나, 파일을 포커스하거나, 결과를 특정 worktree에 기록하는 작업을 CLI 명령으로 수행할 수 있습니다.

## 오픈소스 공개 배경

stablyai는 YC W22 출신 스타트업으로, 공동창업자 Jinjing Liang과 Neil Parker가 설립했습니다. 샌프라시스코 기반의 소규모 팀(4명)이지만 MIT 라이선스로 전체 코드를 공개했습니다.

오픈소스 전략의 의도는 에이전트 생태계와 함께 성장하는 것입니다. 어떤 에이전트가 뜨더라도 Orca가 그 에이전트를 지원하면 사용자가 자연스럽게 Orca로 모입니다. 특정 에이전트에 락인되지 않는 구조 자체가 경쟁력입니다.

2026년 7월 17일 기준 GitHub 스타 21,702개는 출시 이후 빠른 커뮤니티 반응을 보여줍니다. AI 코딩 툴 중에서도 에이전트 오케스트레이션에 특화된 포지셔닝이 차별화 포인트로 작동한 것으로 보입니다.

## 설치와 시작

```bash
# Homebrew (macOS)
brew install stablyai/tap/orca

# npm
npm install -g @stablyai/orca

# 직접 실행
orca
```

처음 실행하면 사용할 에이전트를 선택하고 API 키를 연결합니다. 이후 프로젝트 디렉터리에서 `orca`를 실행하면 워크트리 분기 인터페이스가 열립니다.
