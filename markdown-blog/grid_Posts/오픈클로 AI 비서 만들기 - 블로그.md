2026년 4월 4일, 13만 명이 넘는 개발자들이 갑자기 청구서를 받았습니다.

Anthropic이 Claude 구독자의 서드파티 앱 접근을 차단했고, 일부 사용자는 기존 대비 50배가 넘는 비용을 마주했습니다. 차단된 앱 중 하나가 오픈클로(OpenClaw)였습니다. GitHub 스타 24만 7천 개. 가장 인기 있는 오픈소스 AI 에이전트 프레임워크 중 하나였습니다.

이 사태는 단순한 가격 논란이 아닙니다. 외부 도구에 내 AI 워크플로우를 올려두는 것의 위험성을 드러낸 사건입니다.

---

## 오픈클로 사태: 무슨 일이 있었나

오픈클로는 Claude API를 중간에서 연결해주는 레이어입니다. Claude 구독 한도 안에서 자동화 에이전트를 돌릴 수 있게 해줬고, 그래서 인기가 많았습니다.

Anthropic의 입장에서는 월 20달러 구독자가 API 가격으로 수백 달러어치를 쓰는 구조였습니다. 결국 4월 4일, 정오를 기점으로 차단이 시행됐고, 오픈클로 사용자들은 추가 요금 청구 또는 서비스 중단 중 하나를 선택해야 했습니다.

여기서 핵심 질문이 나옵니다.

"처음부터 내가 제어하는 시스템으로 만들었다면?"

---

## 외부 프레임워크 의존의 구조적 문제

오픈클로 사태는 예외가 아닙니다. AI 플랫폼의 정책이 바뀔 때마다 같은 일이 반복됩니다.

- 정책 변경 → 사용자 공지 없이 기능 차단
- 가격 인상 → 기존 워크플로우 비용 폭등
- 서비스 종료 → 의존하던 자동화 전체 중단

외부 프레임워크에 올린 AI 워크플로우는 그 플랫폼의 생존에 달려 있습니다. 스타트업이 피벗하거나 대기업이 정책을 바꾸면, 내 시스템이 함께 흔들립니다.

반면, Claude Code + 스킬로 구성한 시스템은 내 환경에 직접 설치됩니다. Claude API를 직접 쓰거나, 스킬을 통해 자동화하는 구조이기 때문에 중간 레이어가 없습니다.

---

## Claude Code + 스킬의 구조

Claude Code는 터미널에서 돌아가는 AI 어시스턴트입니다. 단순한 챗봇과 다른 점은 내 파일 시스템, git, 터미널 명령어에 직접 접근한다는 것입니다.

여기에 스킬 시스템을 더하면 AI 비서가 됩니다.

```
.claude/
  settings.json      ← 하네스(권한·훅·MCP 서버 설정)
  skills/
    blog-draft/      ← 블로그 초안 자동화
    telegram/        ← 텔레그램 알림 자동화
    youtube-finder/  ← 유튜브 주제 발굴 자동화
```

스킬은 SKILL.md 파일에 쓴 워크플로우 명세입니다. Claude Code가 이 파일을 읽고, 트리거 조건이 맞으면 자동으로 그 흐름을 실행합니다.

오픈클로가 해주던 것 — "특정 상황에서 자동으로 AI가 연속 작업을 처리" — 을 외부 서버 없이, 내 로컬 환경에서 구현하는 방식입니다.

---

## 실전 구성: MCP + 훅 + 스킬

AI 비서를 구성하는 세 가지 요소입니다.

**MCP 서버** — 외부 도구 연결

Model Context Protocol. Claude가 외부 서비스와 대화하는 방법입니다. Linear, Notion, Figma, Telegram, Zotero 등을 Claude에 연결합니다. 설정은 settings.json에 합니다.

```json
{
  "mcpServers": {
    "linear": { ... },
    "telegram": { ... }
  }
}
```

**훅(Hooks)** — 이벤트 기반 자동화

Claude Code가 특정 동작을 할 때 자동으로 실행되는 셸 스크립트입니다. "파일 저장 후 git commit 자동 실행", "작업 완료 후 텔레그램 알림 전송" 같은 것들입니다.

**스킬** — 반복 워크플로우 패키징

매번 설명하지 않아도 되는 "내가 자주 하는 일"을 스킬로 만들어두면, 트리거 한 번으로 전체 워크플로우가 실행됩니다.

세 가지를 조합하면 오픈클로보다 내 환경에 맞는 AI 비서가 됩니다.

---

## 지속 가능한 AI 비서의 조건

AI 비서를 만들 때 가장 먼저 물어야 할 것은 "이게 얼마나 오래 갈 수 있는가"입니다.

- 외부 서비스에 의존하는 부분은 최소화할수록 좋습니다.
- 핵심 데이터(메모, 문서, 지식베이스)는 내가 소유하는 형식(마크다운, JSON)으로 저장합니다.
- 스킬과 훅은 버전 관리(git)로 추적합니다.

오픈클로 사태가 준 교훈은 "플랫폼이 바뀌어도 내 워크플로우는 살아있어야 한다"는 것입니다.

Claude Code + 스킬 시스템은 그 조건을 맞출 수 있는 구조입니다. 설정 파일과 스킬 파일이 모두 내 로컬에 있고, Claude API를 직접 쓰는 구조이기 때문에 중간 레이어가 끊겨도 재구성이 가능합니다.
