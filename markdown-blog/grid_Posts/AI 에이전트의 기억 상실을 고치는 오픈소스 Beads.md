AI 에이전트에게 장기 작업을 맡겨보신 적 있으신가요? 세션을 새로 열면 이전에 내린 결정, 진행 상황, 맥락을 처음부터 다시 설명해야 합니다. 에이전트가 기억을 잃기 때문입니다.

이 문제를 해결하려고 만들어진 오픈소스 프로젝트가 있습니다. 이름은 **Beads**입니다. GitHub에서 이미 22,600개 이상의 스타를 받았습니다.

> 원문 참고: https://x.com/vista8/status/2049651974317191464

## 왜 Markdown으로는 안 되는가

현재 대부분의 AI 에이전트는 Markdown을 기억 저장 수단으로 씁니다. CLAUDE.md나 Memory.md 같은 파일에 지시 사항과 맥락을 적어두는 방식입니다.

Markdown은 순수 텍스트입니다. 구조가 없고, 항목 간 의존 관계를 표현할 수 없으며, 상태 추적이 불가능합니다. 작업이 많아지면 파일이 길어지고, 컨텍스트 창이 가득 차면 정보가 그냥 사라집니다. 에이전트는 모릅니다. 사용자도 언제 정보가 사라졌는지 모릅니다.

Beads는 이 구조 자체를 바꾸려고 합니다.

## Dolt: Git 같은 SQL 데이터베이스

Beads의 기반 기술은 **Dolt**입니다. Dolt는 SQL 데이터베이스인데, Git처럼 작동합니다. 브랜치, 병합, 버전 되돌리기를 지원하고, 심지어 셀 단위의 병합까지 가능합니다.

여기에서 Beads가 얻는 구체적인 장점이 세 가지입니다.

첫째, **동시 쓰기 충돌 방지**입니다. 여러 에이전트가 동시에 같은 데이터베이스에 쓰기 작업을 할 때, 해시 ID(예: `bd-a1b2`)를 사용해서 충돌 없이 병렬 작업이 가능합니다.

둘째, **작업 이력 보존**입니다. 모든 작업이 버전으로 기록됩니다. 무언가 잘못됐을 때 특정 시점으로 되돌아갈 수 있고, 작업이 어떤 순서로 진행됐는지 추적할 수 있습니다.

셋째, **원격 동기화**입니다. 팀 협업이나 여러 기기 사용 시에도 상태를 동기화할 수 있습니다.

Git에는 의존하지 않습니다. Beads는 Git 없이 완전히 독립적으로 실행됩니다.

## 의미 기억 감쇠: 컨텍스트 창을 절약하는 법

Beads에는 **의미 기억 감쇠(Semantic Memory Decay)** 라는 기능이 있습니다. 완료된 작업을 압축 요약해서 저장하는 방식입니다.

완료된 태스크의 전체 대화 내용을 컨텍스트 창에 계속 유지할 필요가 없습니다. 핵심만 압축한 요약으로 대체하면 창 공간을 크게 절약할 수 있습니다. 현재 진행 중인 작업에 더 많은 컨텍스트를 쓸 수 있게 됩니다.

## 어떤 상황에서 쓸 수 있는가

Beads의 주요 대상은 AI 프로그래밍 에이전트입니다. 코드 작업처럼 여러 세션에 걸쳐 진행되고, 중간 결정이 누적되는 작업에서 효과가 큽니다.

하지만 이 접근 방식은 더 넓게 적용됩니다. 여러 AI 세션 사이에서 작업 연속성을 유지해야 하는 모든 시나리오 — 연구 프로젝트, 장기 문서 작업, 반복적인 분석 작업 — 에도 쓸 수 있습니다. 매번 새 세션을 열 때마다 컨텍스트를 처음부터 설명할 필요가 없어집니다.

## 설치 방법

GitHub 주소를 에이전트에게 주고 설치를 요청하는 방식으로 시작할 수 있습니다. 설정을 에이전트에게 설명하고, Beads를 기억 저장소로 활용하도록 안내하면 됩니다. Git 환경이 없어도 독립적으로 실행됩니다.

AI 에이전트를 장기 작업에 활용하고 싶은데 세션 단절이 병목이었다면, Beads는 구조적인 해결책이 될 수 있습니다.
