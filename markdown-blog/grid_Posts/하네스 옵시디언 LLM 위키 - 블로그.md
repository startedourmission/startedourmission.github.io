지식 관리의 가장 큰 문제는 입력하는 것보다 찾는 것이 더 어렵다는 점입니다.

수백 개의 노트를 쌓아두고도 필요한 순간에 연결하지 못합니다. 정보가 많은데 정작 "저번에 읽은 그 논문"은 어느 폴더에 있는지 기억나지 않습니다.

LLM 위키는 이 문제를 다른 방식으로 풉니다. AI가 노트를 생성하고, 연결하고, 유지보수합니다. 사람은 입력과 검색만 합니다.

---

## 하네스란 무엇인가

Claude Code에서 하네스(Harness)는 `.claude/settings.json`으로 관리하는 환경 설정입니다. AI가 무엇을 할 수 있고, 어떤 도구에 접근하고, 어떤 이벤트에 반응하는지를 정의합니다.

구체적으로는 세 가지를 담습니다.

**권한(Permissions)** — Claude가 실행할 수 있는 명령어 범위를 지정합니다. `Bash(git commit:*)` 같은 패턴으로 안전하게 허용합니다.

**훅(Hooks)** — Claude의 특정 동작(파일 저장, 작업 완료, 메시지 전송)에 연결되는 셸 스크립트입니다.

**MCP 서버** — 외부 서비스(Zotero, Linear, Telegram, Obsidian 플러그인)를 Claude에 연결합니다.

이 세 가지를 설정한 환경이 "하네스"입니다. Claude Code가 그냥 AI 챗봇이 아니라 자율 에이전트처럼 동작하게 하는 기반입니다.

## Obsidian이 LLM 위키의 최선인 이유

지식 관리 도구는 많습니다. Notion, Roam Research, Logseq, Bear. 그중 Obsidian이 LLM 위키 구축에 가장 적합한 이유는 두 가지입니다.

**마크다운 파일 기반** — Obsidian은 내 컴퓨터에 저장된 `.md` 파일들을 보여주는 뷰어입니다. 파일 자체가 데이터입니다. Claude Code는 마크다운 파일을 직접 읽고 쓸 수 있습니다. 별도 API 없이 연결됩니다.

**위키 링크 시스템** — `[[파일명]]` 형식의 링크로 노트 간 연결을 만듭니다. Claude가 노트를 생성할 때 관련 노트와의 링크를 자동으로 추가할 수 있습니다. 사람이 수동으로 연결하지 않아도 됩니다.

Notion 같은 클라우드 기반 도구는 API를 통해서만 접근 가능합니다. 속도가 느리고 API 한도가 있습니다. Obsidian은 파일 시스템을 직접 씁니다.

## LLM 위키의 구조

AutoVault(이 블로그의 Obsidian 볼트) 구조를 예시로 씁니다.

```
AutoVault/
  raw/                  ← 원시 입력
    clippings/          ← 웹 클리핑
    drafts/             ← 작업 중인 문서
  output/               ← 빌드 산출물
  블로그/               ← 퍼블리싱 공간 (git repo)
  Clippings/            ← 처리된 클리핑
  _Templates/           ← Obsidian 템플릿
  .claude/
    memory/             ← Claude 장기 메모리
    skills/             ← 자동화 워크플로우
    settings.json       ← 하네스 설정
```

Claude는 이 구조를 알고 있습니다. CLAUDE.md 파일에 볼트 구조, 규칙, 워크플로우를 기록해두면 Claude가 항상 그 맥락을 유지합니다.

LLM 위키의 핵심은 Claude의 장기 메모리입니다. `.claude/memory/` 폴더에 사용자 프로필, 프로젝트 상태, 피드백 이력을 저장합니다. 다음 대화에서도 이전 맥락을 유지합니다.

## 실전 구축 단계

4단계로 구성합니다.

**1단계: 볼트 구조 설계**

어떤 정보를 어디에 저장할지 먼저 정합니다. 블로그 초안, 리서치 노트, 웹 클리핑, 프로젝트 문서를 구분하는 폴더 구조를 만듭니다. CLAUDE.md에 이 구조를 문서화합니다.

**2단계: 하네스 설정**

`.claude/settings.json`에 필요한 도구들을 연결합니다. 글쓰기 워크플로우라면 git 권한, Zotero MCP, Telegram 알림이 기본 구성입니다.

**3단계: 스킬 설치**

반복되는 작업을 스킬로 만듭니다. 웹 클리핑 → 요약 → Obsidian 저장 같은 흐름을 하나의 스킬로 만들면 `/clip-save URL`로 한 번에 실행됩니다.

**4단계: 메모리 시스템 구성**

Claude가 볼트를 사용하면서 배운 것들을 `.claude/memory/` 에 저장합니다. 프로젝트가 진행될수록 Claude의 맥락 이해가 깊어집니다.

## 자동 유지보수로 확장하기

LLM 위키의 가장 강력한 기능은 자동 유지보수입니다.

새로운 노트가 생길 때 기존 노트와의 연결을 Claude가 찾아줍니다. 중복 내용이 있으면 통합을 제안합니다. 오래된 정보는 업데이트가 필요한지 알려줍니다.

이것이 기존 지식 관리 도구와 다른 점입니다. 기존 도구는 정보를 잘 보관합니다. LLM 위키는 정보가 살아있게 유지합니다.

처음 구축에 며칠이 걸리지만, 이후에는 Claude와 함께 자라나는 지식 시스템이 됩니다.
