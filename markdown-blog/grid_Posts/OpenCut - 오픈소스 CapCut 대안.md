CapCut을 쓰기 불편해진 이유가 몇 가지 쌓였습니다. 2025년 1월 TikTok과 함께 미국 앱스토어에서 24시간 퇴출됐다가 복귀했고, 아일랜드 데이터보호위원회(DPC)는 2025년 5월 GDPR 위반으로 5억 3천만 유로 벌금을 물렸습니다. 같은 해 6월 업데이트된 이용약관은 사용자가 올린 영상에 "영구적이고 취소 불가능하며 로열티 없는 재사용 권한"을 ByteDance에 부여합니다.

OpenCut은 이 맥락에서 GitHub 트렌딩 1위에 올랐습니다. MIT 라이선스, 68,838 스타(2026-07-14 기준).

## CapCut이 대체 타겟이 된 이유

데이터 우려는 어제오늘 일이 아닌데 왜 지금 OpenCut이 터졌냐면, 그 우려가 숫자로 현실화됐기 때문입니다.

2025년 1월 미국에서 TikTok·CapCut이 앱스토어에서 삭제됐을 때, 수천만 명의 크리에이터가 편집 도구를 갑자기 잃었습니다. 트럼프 행정부가 집행을 연기하면서 하루 만에 복귀했지만, 이 사건은 "언제든 다시 벌어질 수 있다"는 인식을 남겼습니다. DPC의 €530M 벌금과 약관 개정이 그 위에 겹쳤습니다.

크리에이터 입장에서 이 상황이 불편한 이유는 단순히 "중국 앱이 불안하다"는 감정이 아닙니다. 편집 작업물과 소스 영상을 특정 플랫폼에 올릴 때 그 회사의 약관 변경에 종속된다는 구조적 문제입니다.

## OpenCut의 구조

OpenCut은 두 갈래로 나뉩니다. [opencut.app](https://opencut.app)에서 지금 바로 쓸 수 있는 클래식 에디터와, GitHub 기본 브랜치에서 진행 중인 ground-up 재작성 버전입니다.

재작성 버전의 설계 방향이 흥미롭습니다.

**Rust 코어.** GPU 합성, 이펙트, 마스크 처리를 Rust로 구현합니다. 이 Rust 코어를 WebAssembly로 컴파일해 브라우저에서 네이티브에 가까운 성능을 냅니다. 동시에 동일한 코드가 데스크탑 앱(GPUI 기반)에서도 실행됩니다. 웹/데스크탑/모바일이 같은 렌더링 파이프라인을 씁니다.

**플러그인 우선 아키텍처.** 기능을 모놀리식으로 쌓지 않고, 각 기능이 플러그인 형태로 붙습니다. 커뮤니티가 새 이펙트나 내보내기 형식을 독립적으로 추가할 수 있는 구조입니다.

**MCP + 헤드리스 API 계획.** 로드맵에 MCP(Model Context Protocol) 서버 지원과 헤드리스 실행 API가 포함돼 있습니다. MCP 연동이 완성되면 Claude Code 같은 도구에서 편집 명령을 보내고 OpenCut이 이를 실행하는 흐름이 가능해집니다. "AI가 시나리오를 짜면 OpenCut이 영상을 만든다"는 파이프라인의 실행 엔진 역할입니다.

## 오픈소스 영상 편집 생태계에서의 위치

기존 오픈소스 영상 편집기들은 쓰기 어렵다는 평을 오래 받았습니다.

Kdenlive와 OpenShot은 Linux 중심의 전통적인 타임라인 편집기입니다. 기능은 충분하지만 UI가 CapCut의 모바일·웹 경험과 거리가 멉니다. DaVinci Resolve Free는 기능면에서 가장 강력하지만, 무료 버전에도 클라우드 협업·일부 AI 기능이 막혀 있고, Blackmagic Design이 소스를 공개하지 않습니다.

OpenCut은 다른 포지션을 겨냥합니다. CapCut처럼 빠르고 웹 기반이면서, MIT 라이선스로 약관 변경 리스크가 없습니다. Vercel과 fal.ai의 지원을 받고 있고, 90명 이상의 컨트리뷰터가 붙었습니다.

클래식 에디터는 지금 당장 쓸 수 있고, 재작성 버전은 기술적 기반을 제대로 닦는 중입니다. 아직 CapCut의 AI 자막 생성, 자동 편집 템플릿 같은 기능을 대체하기는 어렵습니다. MCP 헤드리스 API가 열리면 그 격차를 AI 도구 연동으로 메울 수 있는 구조입니다.
