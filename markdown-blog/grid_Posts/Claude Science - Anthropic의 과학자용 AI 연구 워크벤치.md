[[Anthropic]]이 2026년 6월 30일 공개한 Claude Science는 과학 연구자를 위한 에이전트 기반 워크벤치입니다. [[Claude]]를 과학 연구 워크플로에 직접 연결한다는 개념인데, 단순히 질문에 답하는 챗봇이 아니라 코드 실행, 데이터 분석, 논문 초안 생성까지 이어지는 멀티에이전트 파이프라인으로 구성되어 있습니다.

현재 macOS와 Linux에서 베타로 제공되며, Pro, Max, Team, Enterprise 유료 플랜에서 사용할 수 있습니다.

## 멀티에이전트 구조

Claude Science의 핵심은 코디네이터 에이전트 + 전문 스킬 조합입니다. 코디네이터는 사용자의 연구 목표를 파악하고, 60개 이상의 curated skill과 connector 중에서 필요한 것을 골라 호출합니다. 유전체학, 단세포 분석, 단백질체학, 구조생물학, 화학정보학 분야의 스킬이 포함되어 있고, 사용자가 자신만의 전문 에이전트를 직접 추가하는 것도 가능합니다.

리뷰어 에이전트가 별도로 작동합니다. 본문의 인용문, 계산값, 그림이 실제 코드 결과와 일치하는지를 자동으로 점검하고 불일치가 발견되면 자기교정을 시도합니다. 논문 초안이나 리포트에서 수치가 틀린 채로 나오는 실수를 줄이기 위한 장치입니다.

## 생성 산출물

Claude Science가 만들어내는 산출물은 코드에 그치지 않습니다. 3D 단백질 구조, 게놈 브라우저 트랙, 화학 구조 등 분야별 시각화를 직접 렌더링하고, 코드와 함께 그림, 논문 초안까지 한 번에 생성할 수 있습니다.

기존 AI 도구들이 텍스트 출력에 머무르던 것과 비교하면 단계가 다릅니다. 분석 코드를 실행하고, 결과를 시각화하고, 그 위에 논문 서술까지 자동으로 이어진다면 연구자가 손으로 하던 반복 작업 상당 부분이 줄어듭니다.

## 컴퓨트 관리

노트북, 클러스터, GPU 온디맨드 환경 모두에서 실행됩니다. 민감한 연구 데이터를 외부 서버로 보낼 필요 없이 로컬 환경이나 기관 클러스터에서 작업할 수 있습니다. 임상 데이터나 미공개 데이터를 다루는 연구실에서는 이 부분이 중요한 조건입니다.

파트너십으로는 NVIDIA BioNeMo Agent Toolkit을 연동했습니다. 바이오인포매틱스 워크로드를 가속하는 NVIDIA의 플랫폼과 연결해 구조 예측, 분자 생성 등 GPU 집약적 작업을 처리하는 방식입니다.

## 초기 사용 사례

공개 발표에 함께 이름을 올린 사용자가 세 팀입니다.

Manifold Bio는 조직 표적 의약품 후보 바인더 수백만 개를 대상으로 표면 발현, 트래피킹, 안전성을 평가해 실험에 올릴 타깃을 선정하는 데 Claude Science를 사용했습니다. 후보 규모가 커 사람이 일일이 우선순위를 매기기 어려운 작업입니다.

Allen Institute의 신경과학자 Jerome Lecoq는 약 20개의 커스텀 스킬로 멀티에이전트 리뷰 템플릿을 구성해 문헌 리뷰를 자동 생성하는 데 적용했습니다. 100쪽이 넘는 리뷰를 여러 편 만들었고, 기존에 2년 가까이 걸리던 작업을 크게 단축했다고 밝혔습니다. Allen Institute는 뇌 지도 프로젝트와 오픈 데이터셋으로 잘 알려진 비영리 연구기관입니다.

UCSF Brain Tumor Center의 Stephen Francis는 종양 연구에 활용해 germline 분석을 기존의 10분의 1 시간에 처리했다고 밝혔습니다. 임상 데이터를 다루는 맥락에서 로컬 환경 실행이 가능하다는 점도 함께 작용했습니다.

## AI for Science Grant Program

동시에 AI for Science Grant Program도 공개했습니다. 50개 프로젝트를 선정해 프로젝트당 30,000달러 상당의 Claude API 크레딧을 지원하는 프로그램입니다. 신청 마감은 2026년 7월 15일입니다.

지원 대상은 명시되지 않았지만 연구기관, 대학, 바이오테크 스타트업 등이 주요 대상이 될 것으로 보입니다. 크레딧 지원이라 현금이 아닌 API 사용량으로 주어지며, Claude Science를 실제 연구에 써본 팀을 조기에 확보하려는 의도가 있습니다.

## 현실적 위치

Claude Science는 현시점에서 베타이고 macOS/Linux로만 한정됩니다. 60개 스킬이 curated 형태라는 것은 특정 분야 외의 연구자에게는 기성 스킬이 맞지 않아 직접 에이전트를 구성해야 할 수 있다는 뜻이기도 합니다.

그럼에도 방향은 분명합니다. LLM이 과학 연구 사이클에 직접 들어오는 것입니다. 논문 작성 보조나 코드 생성 수준이 아니라, 실험 설계부터 결과 검증까지 파이프라인 전체에 에이전트가 엮이는 구조입니다. 이 방향의 실제 효과는 결국 벤치마크가 아닌 실험실 생산성 수치로 검증될 것입니다.
